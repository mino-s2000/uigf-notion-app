# プロジェクト構造の整理と仮想環境の構築計画

プロジェクトのルートディレクトリを整理し、開発環境をカプセル化（仮想環境化）することで、よりクリーンな管理を可能にします。

## 変更の概要

1.  **SrcLayout の採用**: すべてのソースコード (.py) を `src/` ディレクトリに移動します。
2.  **仮想環境 (venv) の導入**: プロジェクト専用のライブラリ実行環境を作成します。
3.  **パス管理の修正**: ソースコードが一段深いディレクトリに移動することに伴い、`.env` や JSON ファイルへのパス指定を動的に調整します。

## 詳細な変更内容

### 1. フォルダ構成の変更

以下の構成に変更します：

```
uigf-notion-app/
├── .env
├── .gitignore
├── requirements.txt
├── .venv/ (NEW)
├── src/ (NEW)
│   ├── constants.py
│   ├── notion_api.py
│   ├── utils.py
│   ├── uigf_to_notion.py
│   └── (その他の .py ファイル)
├── docs/
└── (その他のデータファイル)
```

### 2. コード修正

#### [MODIFY] [constants.py](file:///c:/Git/uigf-notion-app/constants.py)

- `.env` ファイルを探す場所を、スクリプトのディレクトリから「一つ上の親ディレクトリ」に変更します。

#### [MODIFY] [utils.py](file:///c:/Git/uigf-notion-app/utils.py)

- キャッシュファイル (`uigf_cache.json`) などの保存場所を、プロジェクトルートに保つか、あるいは `src` 外の適切な場所に誘導するようにパスを調整します。

### 3. 環境構築

- `python -m venv .venv` を実行し、パッケージをインストールします。

## 検証計画

### 動作確認

- 仮想環境を有効化した状態で `src/uigf_to_notion.py` を実行し、正しく動作することを確認。
- `.env` の設定が正しく読み込まれているかを確認。
